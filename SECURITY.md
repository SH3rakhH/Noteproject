# Security Policy — Zeero заметки

Этот документ описывает базовые меры безопасности приложения «Zeero заметки» и правила ответственного раскрытия уязвимостей.[web:464][web:467]

## Overview

- Zeero заметки — локальное десктоп‑приложение для заметок (Python, Tkinter, ttkbootstrap, SQLite).
- Все данные заметок хранятся в локальной базе `notes.db` рядом с исполняемым файлом или в пользовательском каталоге (в зависимости от установки).
- Конфигурационный файл PIN (`pin.cfg`) хранится в пользовательской папке (`%APPDATA%\ZeeroNotes\pin.cfg` на Windows).[web:456][web:459][web:460]

Приложение не отправляет данные заметок в сеть и не использует удалённые API.[web:473]

## Data Storage & Privacy

- Заметки, категории, теги и история версий хранятся в локальной базе SQLite (`notes.db`).
- PIN для доступа к приложению хранится в текстовом виде в файле `pin.cfg` в пользовательском профиле (`%APPDATA%\ZeeroNotes`).[web:456][web:459]
- Ответственность за защиту файлов `notes.db` и `pin.cfg` (шифрование диска, резервные копии, права доступа) лежит на пользователе/администраторе системы.[web:450][web:454]

Рекомендуется:
- использовать учётную запись Windows с паролем;
- включить шифрование диска (BitLocker или аналог);
- не хранить `notes.db` на общедоступных сетевых ресурсах без дополнительных мер защиты.[web:446][web:450]

## Access Control

- Доступ к приложению осуществляется по PIN‑коду, который проверяется при запуске.
- Значение PIN не шифруется и сохраняется в локальном конфигурационном файле, поэтому его следует выбирать и хранить осознанно.
- Для дополнительной защиты рекомендуется:
  - не использовать тот же PIN, что у банков/почты;
  - ограничить физический доступ к рабочей станции.[web:450][web:454]

В будущих версиях возможно добавление:
- шифрования базы данных заметок;
- более сложной схемы аутентификации (например, интеграция с учётной записью ОС).[web:470][web:472]

## Dependencies & Updates

Проект использует сторонние библиотеки:

- Python 3.x
- Tkinter / ttkbootstrap (оформление интерфейса)
- SQLite (локальное хранение данных)[web:343][web:207]

Рекомендуется:
- периодически обновлять Python и зависимости, чтобы получать исправления безопасности;
- пересобирать exe/инсталлятор после обновления библиотек.[web:470][web:473]

## Reporting a Vulnerability

Если вы обнаружили потенциальную уязвимость в Zeero заметки:

- Пожалуйста, не публикуйте подробности в открытых issues или публичных каналах до исправления.[web:465][web:467]
- Отправьте приватное сообщение владельцу проекта (например, по адресу электронной почты, указанному в README) с:
  - кратким описанием проблемы;
  - шагами для воспроизведения;
  - ожидаемым и фактическим результатом;
  - версией приложения (`APP_VERSION`) и ОС.[web:465][web:469]

Мы постараемся:
- подтвердить получение сообщения в разумные сроки;
- оценить важность уязвимости;
- выпустить исправление или дать рекомендации по обходу.

## Scope & Limitations

- Zeero заметки не предназначено для обработки строго конфиденциальных данных (например, государственных тайн, платёжных реквизитов в явном виде).
- Приложение не выполняет автоматическое шифрование заметок и не обеспечивает сетевую безопасность, так как не работает через интернет.
- Пользователь должен самостоятельно позаботиться о резервном копировании и шифровании, если это требуется политикой безопасности его организации.[web:470][web:473]
